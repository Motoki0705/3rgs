# Data Structure

## Overview

This repository expects each scene under `data/{scene}`.
The only required input before preprocessing is `data/{scene}/images/`.

```text
3rgs/
├── data/
│   └── {scene}/
│       ├── images/
│       ├── images_train.txt
│       ├── images_test.txt
│       ├── pose_gt_train.npy
│       ├── pose_gt_test.npy
│       ├── images_4/
│       ├── mast3r/
│       └── mast3r_sfm/
└── third_party/mast3r/
```

## Required Input

Place raw RGB images here:

```text
data/{scene}/images/
```

Example:

```bash
mkdir -p data/tennis_court/images
cp /path/to/source_images/* data/tennis_court/images/
```

## Files Generated by `scripts/preprocess.py`

`scripts/preprocess.py --scene_dir data/{scene}` creates:

- `images_train.txt`
  Train split image stems.
- `images_test.txt`
  Test split image stems.
- `pose_gt_train.npy`
  Reference train poses packaged from MASt3R output.
- `pose_gt_test.npy`
  Reference test poses packaged from MASt3R output.
- `mast3r/camera_intrinsics.npy`
  Train intrinsics used by the MASt3R dataset loader.
- `mast3r/camera_poses.npy`
  Train camera poses.
- `mast3r/pointcloud.ply`
  Sparse point cloud.
- `mast3r/corr_i.npy`
  Encoded correspondence coordinates for the left image in each pair.
- `mast3r/corr_j.npy`
  Encoded correspondence coordinates for the right image in each pair.
- `mast3r/corr_mask.npy`
  Validity mask for each correspondence slot.
- `mast3r/corr_weight.npy`
  Correspondence weights.
- `mast3r/corr_batch_idx.npy`
  Batch index tensor expected by 3RGS.
- `mast3r/corr_is_manual.npy`
  Manual correspondence flags.
- `mast3r/ei.npy`
  Left image indices for each training pair.
- `mast3r/ej.npy`
  Right image indices for each training pair.
- `mast3r/depthmaps.npy`
  Placeholder depth tensor used by the current pipeline.
- `mast3r/preprocess_summary.json`
  Summary of train/test counts and pair counts.
- `mast3r_sfm/`
  Raw MASt3R-SfM run outputs, including viewer assets and caches.

## Notes

- `src/datasets/mast3r.py` reads from `data/{scene}/mast3r/`.
- `data/{scene}/images_{factor}/` is created lazily by training or epipolar preprocessing.
- `mast3r_sfm/` is an intermediate artifact directory and can be regenerated.
